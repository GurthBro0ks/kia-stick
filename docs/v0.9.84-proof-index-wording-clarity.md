# KIA-Stick-v0.9.84-proof-index-wording-clarity

This polish clarifies local proof index output so operators can distinguish proof classes without treating the newest filesystem proof as the current accepted pushed baseline.

The index now separates:

- `Latest proof`: newest filesystem proof, which may be incomplete or in progress.
- `Latest accepted pushed closeout proof`: newest `RESULT=PASS`, `PUSHED=yes` closeout proof.
- `Latest accepted pushed closeout commit`: the `COMMIT_SHA` recorded by that closeout proof.
- `Latest operator QA PASS proof`: newest non-closeout proof with manual QA `PASS`.
- `Latest accepted-WARN proof`: parked accepted-WARN evidence, not PASS.
- `Latest screenshot review-ready candidate`: screenshot-gated review candidate.

The current accepted pushed closeout proof must surface commit `bc8fbef3114631ea3e0363b8e700ce0c2dce236e`.
The newest in-progress implementation proof may have `WARN_MISSING_RESULT` until its `RESULT.md` is written, and that WARN remains visible.

This wording change is proof/report/operator UX only. It does not mutate proof directories, read real documents, scan arbitrary folders, approve real-doc work, or add runtime import capability.

Next/PostCSS remains `WARN_SAFE_NEXT_TARGET_UNCLEAR`.
v0.9.12C remains blocked.
queue-015 remains blocked.
Package files remain unchanged.
