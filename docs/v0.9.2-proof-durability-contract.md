## Purpose

This contract defines how KIA Stick handles proof directories that disappear from `/tmp` after a phase has already been reviewed.

This is docs/tests/tooling/state work only. It does not approve real-doc work and does not inspect private folders or user files.

## Contract

- Missing original proof directories are `WARN` unless a replacement proof is explicitly recorded.
- A replacement proof can satisfy the warning only when it records:
  - original missing proof path,
  - replacement proof path,
  - validation status,
  - queue status,
  - push status,
  - `HEAD`/`origin/main` equality status.
- Missing `/tmp` proof must not be treated as `PASS` unless replacement proof is present and validation passed.
- Proof summaries must stay GitHub-safe.
- Proof durability checks may read proof metadata such as `RESULT.md` and directory names under allowed proof roots only.
- Proof durability checks must not copy or inspect private files, APWU paths, user-provided packet files, or real documents.
