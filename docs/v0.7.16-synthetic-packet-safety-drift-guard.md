# KIA Stick v0.7.16 Synthetic Packet Safety Drift Guard

Phase: `KIA-Stick-v0.7.16-synthetic-packet-safety-drift-guard`

Status: synthetic-only guard for validator/report-runner safety drift

Product version: `0.7.0`

Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`

## Scope

This phase adds a local guard helper for the accepted v0.7.14 synthetic approval-packet validator and the accepted v0.7.15 synthetic packet report runner.

The guard reads a fixed allowlist of repo-owned files only. It does not accept path arguments. It does not read user-provided files. It does not scan directories. It does not name, request, read, scan, upload, copy, hash, OCR, index, summarize, redact, inspect, transform, or touch any real document or private source folder.

`queue-015-v07-first-real-doc-gate-request` remains blocked.

## Guard Helper Contract

`scripts/synthetic-packet-safety-guard.mjs` is exposed as:

```bash
npm run packet:guard
```

The helper checks:

- The validator remains synthetic-only and keeps `acceptsPathArguments=false`, `readsUserFiles=false`, and `scansDirectories=false`.
- The report runner remains built-in synthetic fixtures only and keeps `realDocumentAccessed=false`.
- The report runner rejects unexpected positional arguments instead of reading packet files.
- Product version remains `0.7.0`.
- Prompt version remains `prompt.fake-docs.v0.5-import-wizard-hardening`.
- `queue-015-v07-first-real-doc-gate-request` remains blocked.
- `queue-031-v0716-synthetic-packet-safety-drift-guard` records this phase for review.

The helper fails on drift toward file picker, directory picker, path reader, upload handler, OCR, indexing, embedding, vector store, real redaction, real file read, filesystem existence check, directory scan, private source access, or queue-015 unblock behavior.

## GitHub-Safe Output Rules

Allowed guard output:

- Phase name.
- Product and prompt versions.
- Repo-local file IDs.
- Queue IDs and guard booleans.
- Check IDs.
- `PASS`, `WARN`, and `FAIL` labels.

Forbidden guard output:

- Real document names.
- Real snippets.
- Real paths.
- Private source folder names.
- Account/session data.
- Secrets or credentials.
- Hashes from sensitive material.
- OCR output.
- Embeddings or vector data.
- Upload payloads.
- Private notes.

## Rejection Rules

The guard must fail if a caller provides an unexpected positional argument, including a path-shaped argument. It must not inspect that argument, check whether it exists, read it, scan a containing directory, or treat it as an alternate source.

The guard must stay a fixed allowlist drift check. Any future packet-file loader, upload handler, file picker, directory picker, drag/drop import zone, path reader, OCR step, redaction step, text extraction, summarization, embedding, indexing, vector store, APWU/private source access, cloud/API call, service change, or queue-015 transition needs a separate operator-approved phase.
