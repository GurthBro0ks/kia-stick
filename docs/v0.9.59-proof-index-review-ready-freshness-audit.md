# KIA-Stick-v0.9.59-proof-index-review-ready-freshness-audit

Target machine: `USER_LAPTOP_ONLY`

## Audit Result

`npm run proof:index -- latest --root /home/mint/kia-stick-local-proofs` can show a current `Latest proof` while the screenshot-gated review-ready proof remains older.

That is expected when the newest proof directory is an in-progress or closeout proof without `OPEN_THIS_FOLDER.txt` and screenshot filenames. It is confusing only when the output uses one ambiguous review-ready label.

## Safe Metadata Criteria

The proof index is allowed to inspect only proof-safe metadata inside the supplied proof root:

- proof directory names
- nested `closeout_push_YYYYMMDDTHHMMSSZ` and `warn_closeout_push_YYYYMMDDTHHMMSSZ` directory names
- `RESULT.md`
- `OPEN_THIS_FOLDER.txt`
- filenames under a proof-local `screenshots/` directory

It must not read screenshots, OCR images, inspect private artifacts, read arbitrary paths, scan outside the supplied proof root, read real documents, summarize proof attachments, or mutate proof directories.

## Review-Ready Meaning

The stricter screenshot-gated candidate still requires:

- `RESULT=PASS`
- `OPEN_THIS_FOLDER.txt`
- at least one screenshot filename under `screenshots/`

Accepted pushed closeout proof is a separate concept. It can be current and accepted without screenshots when safe `RESULT.md` metadata proves `RESULT=PASS`, `MANUAL_QA_STATUS=PASS`, `PUSHED=yes`, and `HEAD == origin/main` was recorded by closeout proof.

## Required Output Labels

The operator should see separate labels for:

- latest proof
- latest accepted pushed closeout proof
- latest operator QA PASS proof
- latest accepted-WARN proof, if any
- latest screenshot review-ready candidate
- why an older screenshot-gated proof is selected, if it is
