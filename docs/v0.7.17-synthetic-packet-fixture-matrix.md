The matrix in `lib/syntheticPacketFixtures.ts` includes:

- `PASS`: complete synthetic packet.
- `WARN`: safe incomplete packet missing reviewer and retention placeholders.
- `FAIL`: path-shaped placeholder.
- `FAIL`: recursive scope wording.
- `FAIL`: broad source wording.
- `FAIL`: private-source wording.
- `FAIL`: queue-015 unblock implication.
- `FAIL`: multiple future document placeholders.
- `FAIL`: multiple future gate placeholders.
- `FAIL`: upload/OCR/indexing/embedding/vector-store capability wording.

The fixture IDs and reason codes are safe for GitHub output. They contain fake labels only and no real document names, private paths, source snippets, OCR text, embeddings, indexes, vector data, secrets, or account/session values.
