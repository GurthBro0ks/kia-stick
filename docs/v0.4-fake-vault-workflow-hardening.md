# KIA Stick v0.4 Fake Vault Workflow Hardening

- Phase: `KIA-Stick-v0.4-fake-vault-workflow-hardening`
- Target: `USER_LAPTOP_ONLY`
- Status: implemented as fake metadata workflow hardening only
- Real document access: none
- Private vault inspected: no
- Push performed: no

## Scope

v0.4 hardens the v0.3 fake-vault scaffold. It keeps the work inside synthetic metadata fixtures, local mock state, tests, and GitHub-safe documentation.

This phase does not read, copy, scan, OCR, index, summarize, transform, upload, or commit real APWU, USPS, member, local, case, steward, or private-vault documents.

## Workflow

The fake lifecycle is now:

```text
selected
-> quarantine
-> hash/provenance
-> redaction review
-> metadata review
-> index eligibility
-> audit
```

Redaction approval and metadata approval are separate mock review actions. Advancing past either gate before the approval exists is blocked and recorded in the audit log.

## Explicit States

The fake workflow exposes these states:

- `not_indexable`
- `quarantine_only`
- `redaction_required`
- `metadata_required`
- `review_rejected`
- `eligible_fake_only`

The only positive eligibility state is `eligible_fake_only`. It means the row is eligible for fake metadata workflow testing, not real indexing.

## Blocked Transitions

Invalid transitions are blocked with visible reasons on the affected fake metadata record and in the audit log.

Examples:

- Metadata review is blocked until redaction review is approved.
- Index eligibility is blocked until metadata review is approved.
- Final audit is blocked unless the state is `eligible_fake_only`.
- Restricted-sensitive fake metadata is rejected by default.

## Audit Export

The Audit Log can export JSON or Markdown. Export payloads include:

- fake metadata rows
- fake audit entries
- `productVersion`
- `displayVersion`
- build date
- Git SHA
- corpus/index/prompt/provider metadata

Export payloads exclude:

- private paths
- file bytes
- source document text
- raw upload content
- account/session data
- real member/local/case details

## Tests

Vitest coverage now checks:

- fake fixtures exist for every lane
- quarantine stays not indexable
- invalid transitions are blocked with reasons
- redaction review and metadata review are required before fake eligibility
- rejected reviews become `review_rejected`
- real path/content payloads are blocked and redacted from audit notes
- JSON and Markdown audit exports include build identity and exclude private paths

## Remaining Boundary

Future phases remain fake-doc/fake-metadata-only unless separately approved. No real-document ingestion, OCR, scanning, indexing, upload workflow, private-vault inspection, or cloud sync is authorized by v0.4.
