# KIA Stick v0.8.1 Queue Reality Audit

Phase: `KIA-Stick-v0.8.1-queue-reality-audit`

Status: backlog reality audit for stale historical queue items

Product version: `0.7.0`

Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`

## Scope

This audit explains why `npm run queue:next` still selected `queue-011-v07-pause-stabilize` after the accepted v0.7.13 through v0.8.0 synthetic governance work was pushed.

The cause was queue metadata drift, not missing accepted work. `scripts/task-queue.mjs` selected the first item whose status was not `accepted`. That old selector treated `planned`, `blocked`, and `parked` as equal candidates. Because `queue-011` remained `planned`, it masked the accepted later sequence and hid the actual next actionable bundle state.

`queue-015` was already correct. It must remain blocked until a separate approval packet explicitly changes that state. This audit does not unblock it.
