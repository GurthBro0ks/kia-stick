## Impact Assessment

`next` is the only direct vulnerable package on the runtime/build path. The npm audit fix suggestion is not directly usable in this phase because it requires force and points to an older major framework version. A safe fix likely requires a separately approved Next upgrade or advisory-specific framework review, followed by full lint, typecheck, test, build, release, and browser smoke validation.

`vitest` is a direct dev/test dependency. Its transitive `vite`, `vite-node`, `@vitest/mocker`, and `esbuild` advisories affect local development/test tooling rather than the shipped fake-doc runtime. A safe fix path likely requires a separately approved Vitest major upgrade from the current v2 line, plus focused test-runner compatibility validation.

## Safe Non-Breaking Fix Assessment

No safe non-breaking fix is applied or proven in this phase.

The audit output reports forced fix paths for both direct vulnerable packages. Because forced fixes can cross major versions or otherwise rewrite dependency resolution, this phase records the risk and stops before package mutation.

Recommended future gates:

1. Review current Next advisory status and identify a supported patched Next version compatible with React 19 and this app.
2. Review Vitest v2-to-v3/v4 compatibility and identify the smallest test-runner upgrade that clears the critical Vitest advisory.
3. Apply dependency changes only in a separately approved implementation phase.
4. Run full validation after any package mutation: `npm run lint`, `npm run typecheck`, `npm test`, `npm run build`, `npm run release:check`, `npm run governance:report`, `npm run queue:next`, and fake-only boundary scans.
