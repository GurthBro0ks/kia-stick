# KIA-Stick-v0.9.20-no-answer-saved-state-qa-hardening

Target machine: `USER_LAPTOP_ONLY`

## Scope

This phase hardens fake-only QA coverage for no-answer and Saved-state behavior without changing runtime capability.

## QA Contract

- No-answer results remain blocked from Saved.
- Repeated saves dedupe by canonical fake answer identity.
- Saved metadata remains fake-only.
- Saved metadata includes product version, prompt version, build/display identity, provider, and citation count when applicable.
- Citable fake source evidence remains distinct from context-only fake source evidence.
- Context-only fake sources can be reviewed in the packet trail but do not make a no-answer result savable.

## Evidence Surfaces

- Chat shows no-answer save blocking and context-only fake source visibility.
- Sources shows citable versus context-only fake source roles.
- Saved shows dedupe, detail metadata, provider, prompt, version, and citation count.
- Settings and `/version` keep the build identity visible for operator review.

## Fake-Only Boundaries

- No real documents are read or summarized.
- No file picker, FileReader, path reader, upload handler, OCR, embeddings, indexing, or vector store is added.
- Product/package version remains `0.7.0`.
- Prompt version remains `prompt.fake-docs.v0.5-import-wizard-hardening`.
- `queue-015-v07-first-real-doc-gate-request` remains blocked.
