# KIA-Stick-v0.9.8-synthetic-governance-report-hardening

Product version: `0.7.0`
Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`
Queue item: `queue-053-v098-synthetic-governance-report-hardening`

## Purpose

This phase hardens synthetic governance report output for operator review and proof-safe closeout.

The reports use built-in synthetic fixtures and fixed repo-owned metadata only. They reject unexpected positional arguments and do not read user-provided files.

## Required Report Signals

- `PASS fixtures`, `WARN fixtures`, and `FAIL fixtures` counts are visible.
- `queue015Status: blocked` is visible.
- `realDocumentAccessed: false` is visible.
- `readsUserFiles: false`, `scansDirectories: false`, and `acceptsPathArguments: false` are visible.
- `stopOnWarnFail: true` is visible.

## GitHub-Safe Output

Report output is limited to fake fixture IDs, PASS/WARN/FAIL labels, reason codes, guard booleans, queue IDs, productVersion, and promptVersion.

Report output must not include raw paths, secrets-looking values, private source names, real document names, OCR output, hashes of real files, copied content, private notes, uploads, embeddings, indexes, or vector data.
