## Goal

This pass makes the Chat tab feel like a real chat surface instead of a database search form. It keeps the same deterministic fake answer governor, source hierarchy, saved answers, upload quarantine UI, Vault guide mode, and build identity.

## Chat Layout

- The current user question renders as a user bubble.
- The KIA Stick response renders as an assistant bubble.
- The first visible answer is compact: short answer, confidence/authority summary, and what to do next.
- The full packet is hidden behind `Show full packet`.
- Citations are hidden behind `Show citations (n)`.
- Prompt shortcuts are tucked under `Prompt shortcuts` so they do not dominate the main workflow.

## Saved Answers

Saved-answer loading now migrates old localStorage entries through the current stable same-chat identity.

- Legacy duplicates with different timestamps or build identities collapse to one saved card.
- Same unchanged saves ignore timestamp and build identity and return `Already saved. No new data.`
- Same-chat saves with changed detail metadata still replace the prior card.
- Different answer context still creates a separate card.

## Source Hierarchy

The Sources tab is grouped by the requested hierarchy:

- Local
- State/Area
- National
- Manuals/Handbooks
- Arbitration/Settlements
- Steward Notes/Evidence
- Unknown

Each source remains fake-bannered and keeps citable/not-citable labels.

## Version Navigation

`/version` now includes a `Back to KIA Stick` button using Next navigation while preserving the full display-version metadata.

This phase stays fake-doc/fake-metadata-only. It does not read, list, copy, scan, OCR, ingest, summarize, transform, upload, index, or commit real APWU, USPS, member, local, case, steward, or private-vault documents.
