# KIA-Stick-v0.9.74-closeout-default-proof-root-discovery-audit

This audit records the closeout-helper default proof-root freshness issue.

Before v0.9.75, `closeout:review` and `closeout:summary` defaulted to `/tmp` unless an operator supplied `--proof-dir` or `--proof-root`. That made default review vulnerable to stale or missing `/tmp` proofs even when persistent KIA proofs existed under `/home/mint/kia-stick-local-proofs`.

Safe discovery order:

1. Explicit `--proof-dir` always wins.
2. `KIA_PROOF_ROOT` wins when supplied and inside an allowed proof root.
3. `/home/mint/kia-stick-local-proofs` wins when it exists and contains KIA proof directories.
4. `/tmp` is fallback only.

The helper must not mutate proof directories. It must not scan arbitrary private folders. Allowed proof roots remain `/tmp` and `/home/mint/kia-stick-local-proofs`.
