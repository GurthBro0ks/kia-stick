# KIA Stick v0.5 Import Wizard Design Plan

## Status

- Phase: `KIA-Stick-v0.5-import-wizard-design-plan`
- Target machine: `USER_LAPTOP_ONLY`
- Target repo: `~/kia-stick`
- Status: planning only
- Baseline: v0.4.1 fake-only release review at `16e1980`
- Product version impact: none
- Real document access: none
- Private vault inspected: no
- Implementation approval: not granted by this plan

This document designs a future real-document import wizard. It does not implement import, file picking, file reads, copying, scanning, OCR, indexing, summarization, transformation, upload handling, or private-vault inspection.

## Planning Boundary

The future wizard must be built as a consent-heavy governance workflow, not as a drag-and-drop upload tool. Every step must keep the operator clear on four rules:

- Selecting a path is not import.
- Quarantine is not indexable.
- Redaction is not approval.
- Approval is not indexing.

No future implementation may treat a prior step as consent for a later step. Each action that touches, copies, derives, reviews, approves, or indexes real content needs its own visible confirmation and audit event.

## Design Goals

- Make the safe path obvious for a steward who is trying to organize documents under pressure.
- Prevent accidental reads, broad folder scans, OCR, embeddings, or indexing.
- Keep raw private material out of Git, proof packs, browser-visible exports, and release artifacts.
- Preserve a local private audit trail while keeping GitHub-safe proof logs sanitized.
- Require human review for redaction, metadata, and index eligibility as separate decisions.
- Make blocked states visible with plain-language reasons.

## Future Wizard Screens

### 1. Start / Safety Warning

Purpose: establish that this is a controlled local workflow and that no content has been touched yet.

Plain copy:

> Import is a review process, not a shortcut. KIA Stick will not read, copy, OCR, summarize, or index anything until you confirm the next step. Start only with a file or small scoped batch you are allowed to review on this laptop.

Required controls:

- `Start planning import`
- `Cancel`
- Required checkbox: `I understand that selection is not import and quarantine is not indexing.`

Stop signs:

- Block if the operator has not acknowledged the safety statement.
- Block if the current phase does not explicitly authorize real-document import implementation.
- Block if privacy scan, Git ignore checks, or local-only settings are not green.

Audit event:

- `wizard_started`
- GitHub-safe fields: wizard version, timestamp, operator alias, safety acknowledgement boolean.
- Never log source paths or filenames.

### 2. Select Source Path Placeholder

Purpose: future UI location for choosing a single file or an explicit scoped batch, without treating selection as a read or copy.

Plain copy:

> Choose what you want to review. This step only points KIA Stick at a candidate. It does not open the document, copy it, read its text, make a preview, or add it to search.

Required controls:

- `Choose one file`
- `Choose scoped batch`
- `Clear selection`
- `Continue to scope confirmation`

Rules:

- Default mode is single-file.
- Batch mode requires explicit included files or an explicit non-recursive scope.
- No recursive folder import.
- No background directory listing outside the operator-selected scope.
- No file content reads, thumbnails, previews, OCR, or metadata extraction in this step.
- UI may show a local-only display alias, not a GitHub-safe path.

Stop signs:

- Block if the selected scope is recursive.
- Block if the selection points to a forbidden or unapproved source area.
- Block if the UI cannot display whether the scope is single-file or explicitly bounded.
- Block if the implementation tries to read bytes during selection.

Audit event:

- `source_candidate_selected`
- Private local audit only may include source alias and operator note.
- GitHub-safe proof may include only selection count, scope type, and a redacted source label such as `operator_selected_single_file`.

### 3. Confirm Single-File Or Explicit Scoped Batch

Purpose: make the operator confirm exact scope before any content is touched.

Plain copy:

> Confirm the scope. KIA Stick can only continue with one file or a small list you explicitly selected. Anything not shown here is out of scope.

Required controls:

- `Confirm one file`
- `Confirm explicit batch`
- `Back to selection`
- `Cancel import`

Rules:

- The confirmation screen must show count, scope type, and lane guess only.
- It must not show raw path strings in GitHub-safe views.
- If batch mode is used, the operator must confirm that no recursive import is happening.
- Confirming scope still does not copy, hash, OCR, summarize, index, or preview content.

Stop signs:

- Block if the count is zero.
- Block if the count exceeds the future configured batch limit.
- Block if any item has unknown scope.
- Block if the operator has not confirmed the non-recursive rule.

Audit event:

- `scope_confirmed`
- GitHub-safe fields: item count, scope mode, batch limit, confirmation timestamp.

### 4. Copy To Quarantine Confirmation

Purpose: make copying raw content into private quarantine a separate consent step.

Plain copy:

> Quarantine is a private holding area. Copying here lets KIA Stick preserve review evidence, but the item is still not redacted, not approved, and not searchable.

Required controls:

- `Copy to quarantine`
- `Cancel before copy`
- Required checkbox: `I understand this creates a private quarantine copy only.`

Rules:

- This is the first future step allowed to read and copy file bytes.
- Quarantine storage must be outside Git-tracked paths.
- Quarantine files are immutable once copied.
- Quarantine files are never index eligible.
- Quarantine files are never exported to GitHub-safe proof packs.
- Any failure must leave the item in a non-imported state with a visible reason.

Stop signs:

- Block if Git ignore checks do not prove quarantine paths are untracked.
- Block if destination path is inside the repo.
- Block if the operator has not confirmed the quarantine-only language.
- Block if the current phase lacks real import authorization.

Audit event:

- `quarantine_copy_requested`
- `quarantine_copy_completed` or `quarantine_copy_failed`
- Private local audit may include quarantine storage ID and raw hash.
- GitHub-safe proof may include only copied count, result status, and sanitized error category.

### 5. Provenance / Hash Step

Purpose: record integrity and source context before any derivative is made.

Plain copy:

> Provenance is the receipt. It records what was copied, when, by whom, and what review lane it starts in. Hashes prove whether the private quarantine copy changed.

Required fields:

- Local document ID
- Operator alias
- Import timestamp
- Starting lane
- Source status
- Sensitivity guess
- Private provenance note
- Raw quarantine hash, private-only
- Copy result

Rules:

- Raw hashes stay private by default.
- GitHub-safe proof must not include raw filenames, raw paths, private notes, or raw content hashes unless a later public-safety review explicitly allows it.
- Provenance cannot approve redaction, metadata, or indexing.

Stop signs:

- Block if the quarantine copy is missing.
- Block if hashing fails.
- Block if required provenance fields are incomplete.
- Block if any provenance field contains raw path dumps, member identifiers, account data, or private narrative content in a GitHub-safe location.

Audit event:

- `provenance_recorded`
- GitHub-safe fields: document ID, lane, source status, sensitivity tier, hash status, no raw hash.

### 6. Redaction Detection Preview

Purpose: show likely sensitive categories before human review, without treating detection as approval.

Plain copy:

> Detection is a warning light. It can miss things and it can over-flag. A person still has to review the document before anything can move forward.

Detection categories:

- Names and aliases
- Addresses, phone numbers, and email addresses
- Employee IDs and other work identifiers
- Medical, leave, restriction, and accommodation details
- Discipline, investigation, grievance, settlement, and witness details
- Case numbers, arbitration numbers, tracking numbers, and file numbers
- Dates or workplace details that could re-identify a matter
- Screenshots, headers, embedded images, metadata, and visible account details

Rules:

- Detection may run only after quarantine copy and provenance are complete.
- Detection output is advisory and cannot mark the item approved.
- Detection previews must be private-local by default.
- GitHub-safe proof may include counts by category, not raw snippets.
- OCR is disabled unless a later implementation phase explicitly approves OCR for the selected item type.

Stop signs:

- Block if quarantine or provenance is incomplete.
- Block if detection tries to write raw snippets to GitHub-safe logs.
- Block if OCR would be required but OCR approval is absent.
- Block if detection output includes unredacted identifiers in a shareable view.

Audit event:

- `redaction_detection_started`
- `redaction_detection_completed`
- GitHub-safe fields: category counts, detection status, no snippets.

### 7. Admin Redaction Review

Purpose: require a human decision on whether the item can become a redacted derivative.

Plain copy:

> Redaction review decides whether a safe derivative can be made. It does not make the item official, searchable, or GitHub-safe.

Review decisions:

- `needs_redaction`
- `approved_redacted_derivative`
- `restricted_sensitive`
- `rejected`
- `more_information_needed`

Rules:

- Reviewer must inspect the private item locally in an approved future implementation, not in GitHub-safe proof.
- Approved redacted derivative must be a new private file with its own hash.
- Raw quarantine copy remains unchanged.
- Restricted-sensitive items default to not indexable.
- Redaction approval cannot skip metadata review.

Stop signs:

- Block if reviewer identity or timestamp is missing.
- Block if detection flags remain unresolved.
- Block if derivative hash is missing when approval claims an approved derivative exists.
- Block if the item is restricted-sensitive and the operator tries to continue toward indexing.

Audit event:

- `redaction_review_decided`
- GitHub-safe fields: decision, reviewer alias, timestamp, reason code, no content.

### 8. Metadata Review

Purpose: separate source authority and governance classification from redaction.

Plain copy:

> Metadata review answers: What kind of source is this, who controls it, how sensitive is it, and how should KIA Stick cite it? This is still not indexing.

Required fields:

- Lane
- Authority level
- Source status
- Sensitivity
- Supersession risk
- Effective date or unknown date
- Distribution limit
- Reviewer alias
- Citation label
- Index recommendation with reason

Rules:

- Metadata review requires redaction review to be resolved first.
- Unknown authority is allowed, but unknown authority blocks positive indexing by default.
- Steward notes must be marked non-controlling.
- Member-only, local-private, steward-private, and restricted-sensitive source statuses must carry stricter defaults than public official sources.
- Metadata cannot store raw paths, member identifiers, case identifiers, or private narrative content in GitHub-safe files.

Stop signs:

- Block if redaction review is missing or rejected.
- Block if authority, source status, sensitivity, or reviewer is missing.
- Block if metadata would reveal a member, case, local matter, or private source path.
- Block if supersession risk is unknown and the operator tries to mark the item controlling.

Audit event:

- `metadata_review_completed`
- GitHub-safe fields: lane, authority level, source status, sensitivity, supersession risk, reviewer alias, no raw source data.

### 9. Index Eligibility Decision

Purpose: make indexing an explicit final gate after redaction and metadata review.

Plain copy:

> Index eligibility decides whether KIA Stick may use the reviewed material for local answers. This is a separate yes/no decision. If there is any doubt, mark it not indexable.

Decision options:

- `not_indexable`
- `eligible_private_local_only`
- `eligible_redacted_local_only`
- `eligible_public_release_candidate`

Rules:

- Default is `not_indexable`.
- Quarantine items are never indexable.
- Raw uploads are never indexable.
- Restricted-sensitive items are never indexable by default.
- Redaction approval alone is not enough.
- Metadata approval alone is not enough.
- Public release candidacy requires a separate future GitHub/public-safety review.
- Positive eligibility may create only a future indexing job record, not immediate embeddings.

Stop signs:

- Block if redaction review is not approved.
- Block if metadata review is not approved.
- Block if the item is in quarantine-only or restricted-sensitive state.
- Block if a positive decision lacks a reason and reviewer alias.
- Block if indexing would run immediately instead of waiting for an explicit future index action.

Audit event:

- `index_eligibility_decided`
- GitHub-safe fields: decision, reason code, reviewer alias, timestamp, no content.

### 10. Audit Summary

Purpose: give the operator a final receipt and produce safe proof options.

Plain copy:

> This summary is a receipt of review decisions. It is not the document. It should prove what gates ran without exposing private content.

Summary sections:

- Scope and item count
- Quarantine copy result
- Provenance status
- Redaction decision
- Metadata decision
- Index eligibility decision
- Blocked transitions and reasons
- Local audit log ID
- GitHub-safe proof eligibility

Rules:

- Default export is GitHub-safe summary only.
- Private local audit is separate and may contain private IDs and hashes.
- GitHub-safe summary must not include raw source paths, filenames if identifying, snippets, OCR text, images, case identifiers, member identifiers, private notes, raw hashes, vector-store data, or account/session data.

Stop signs:

- Block GitHub-safe export if any proof field contains a private path, raw snippet, private note, or identifier.
- Block completion if required audit events are missing.
- Block completion if a positive index decision lacks evidence of separate redaction and metadata review.

Audit event:

- `audit_summary_generated`
- GitHub-safe fields: event counts, final state counts, blocked reason counts, build identity, no content.

## Future State Machine

States:

- `idle_no_selection`: no source candidate.
- `source_candidate_selected`: operator selected a candidate; no bytes read.
- `scope_confirmed`: operator confirmed single-file or explicit bounded batch.
- `awaiting_quarantine_consent`: ready to ask for copy consent.
- `quarantined_not_indexable`: private immutable quarantine copy exists.
- `provenance_required`: copied item needs provenance and hash receipt.
- `provenance_recorded`: private hash and source governance receipt complete.
- `redaction_detection_required`: ready for detection preview.
- `redaction_review_required`: human redaction decision needed.
- `redaction_rejected`: review blocked continuation.
- `redacted_derivative_approved`: private redacted derivative approved.
- `metadata_required`: source governance fields needed.
- `metadata_rejected`: metadata review blocked continuation.
- `metadata_approved`: source governance fields complete.
- `index_decision_required`: explicit yes/no index decision needed.
- `not_indexable`: final or parked state; cannot enter index queue.
- `eligible_private_local_only`: future local-only index queue candidate.
- `eligible_redacted_local_only`: future redacted local-only index queue candidate.
- `eligible_public_release_candidate`: requires separate public-safety review before any commit or publication.
- `audit_complete`: audit receipt generated.

Allowed transitions:

| From | To | Required action |
| --- | --- | --- |
| `idle_no_selection` | `source_candidate_selected` | Operator chooses candidate; no read |
| `source_candidate_selected` | `scope_confirmed` | Operator confirms single-file or explicit bounded batch |
| `scope_confirmed` | `awaiting_quarantine_consent` | System shows quarantine-only warning |
| `awaiting_quarantine_consent` | `quarantined_not_indexable` | Operator confirms copy to private quarantine |
| `quarantined_not_indexable` | `provenance_required` | Copy completes |
| `provenance_required` | `provenance_recorded` | Hash and provenance receipt complete |
| `provenance_recorded` | `redaction_detection_required` | Required fields pass validation |
| `redaction_detection_required` | `redaction_review_required` | Detection preview completes or is manually skipped with reason |
| `redaction_review_required` | `redaction_rejected` | Reviewer rejects or marks restricted |
| `redaction_review_required` | `redacted_derivative_approved` | Reviewer approves derivative |
| `redacted_derivative_approved` | `metadata_required` | Derivative hash and review receipt exist |
| `metadata_required` | `metadata_rejected` | Reviewer rejects metadata |
| `metadata_required` | `metadata_approved` | Required metadata complete |
| `metadata_approved` | `index_decision_required` | Metadata review receipt exists |
| `index_decision_required` | `not_indexable` | Reviewer marks no |
| `index_decision_required` | `eligible_private_local_only` | Reviewer marks local-only private yes |
| `index_decision_required` | `eligible_redacted_local_only` | Reviewer marks redacted local-only yes |
| `index_decision_required` | `eligible_public_release_candidate` | Reviewer marks public candidate; separate review still required |
| Any final decision | `audit_complete` | Audit summary generated |

Blocked transitions:

| Attempt | Block reason |
| --- | --- |
| Select candidate -> quarantine copy without scope confirmation | Selection is not import |
| Select candidate -> hash/provenance | No quarantine copy exists |
| Quarantine -> redaction approval | Detection and human review are missing |
| Quarantine -> index decision | Quarantine is not indexable |
| Redaction detection -> metadata approved | Detection is not approval |
| Redaction approved -> index queue | Approval is not indexing |
| Metadata approved -> index queue without index decision | Metadata is not indexing |
| Any restricted-sensitive state -> positive index eligibility | Restricted-sensitive defaults to not indexable |
| Any state -> GitHub-safe export with raw paths/snippets | Proof would expose private material |
| Any state -> embeddings/vector store without explicit index action | Eligibility is not indexing |

## Stop Signs Before Real Content Is Touched

Future implementation must show and enforce these stop signs before any file bytes are read, copied, OCRed, transformed, summarized, embedded, or indexed:

- Written phase approval explicitly authorizes real import code.
- Operator acknowledges safety warning.
- Operator selects one file or an explicit bounded batch.
- Operator confirms selection is not import.
- Operator confirms scope is non-recursive.
- System proves destination is private and ignored by Git.
- System proves no cloud, network, API, Discord, NUC, SSH, cron, systemd, tmux, or external sync path is active.
- Operator separately confirms copy to quarantine.
- System records a private audit event before and after copy.
- Any failure stops in a visible blocked state.

## Proof Log Rules

Use two proof lanes.

Private local audit may include:

- Local document IDs
- Private source aliases
- Quarantine storage IDs
- Raw quarantine hashes
- Redacted derivative hashes
- Reviewer aliases
- Private operator notes
- Full transition history

GitHub-safe proof may include:

- Command logs
- Validation output
- Git status and changed-file lists
- Counts by state, lane, sensitivity, and decision
- Sanitized reason codes
- Build identity
- Local audit IDs that do not reveal source details
- Redacted derivative hashes only after a separate public-safety review marks them safe to disclose

GitHub-safe proof must never include:

- Raw documents
- Raw filenames when identifying
- Private source paths
- Raw snippets
- OCR text
- Screenshots of documents
- Member names or identifiers
- Case, grievance, arbitration, settlement, witness, medical, discipline, or employment identifiers
- Private notes
- Raw hashes by default
- Vector-store files
- Exports, backups, uploads, or private database files
- Secrets, tokens, cookies, auth headers, account data, or device data

## UI Rules

- Every screen must show the current state and whether content is indexable. Default label: `Not indexable`.
- Destructive or content-touching actions must be verbs: `Copy to quarantine`, `Create redacted derivative`, `Mark eligible`.
- Passive navigation must not use content-touching verbs.
- Buttons that advance gates must show the gate they satisfy.
- Blocked buttons must show a visible reason, not only disable silently.
- Positive status labels must say exactly what is approved: `Redaction approved`, `Metadata approved`, or `Index eligible local-only`.
- Avoid vague labels such as `Ready`, `Imported`, `Approved`, or `Processed` unless the exact gate is named beside them.
- Use plain steward-friendly language first; keep technical fields behind a details toggle.
- The wizard must make it clear that no cloud or external service is involved.
- The wizard must show when the next action would touch real content.

## Future Implementation Acceptance Checklist

Before any future implementation can be accepted:

- Planning-only phase has been superseded by a written implementation phase.
- No real content is touched before explicit operator consent.
- Selection, scope confirmation, quarantine copy, provenance, redaction detection, redaction review, metadata review, index eligibility, and audit summary are separate actions.
- Single-file mode is the default.
- Batch mode is explicitly bounded and non-recursive.
- Quarantine copy is private, ignored by Git, immutable, and never indexable.
- Provenance and raw hashes are private-only by default.
- Detection preview writes no raw snippets into GitHub-safe logs.
- Human redaction review is required before metadata review can approve.
- Human metadata review is required before index eligibility can approve.
- Index eligibility creates a decision record only; it does not immediately build embeddings.
- Restricted-sensitive items default to `not_indexable`.
- All blocked transitions show visible reasons.
- Audit exports split private local audit from GitHub-safe proof.
- GitHub-safe proof contains no raw docs, paths, snippets, OCR, identifiers, private notes, vector stores, exports, backups, uploads, or secrets.
- `git status --short` and private path `git ls-files` checks are clean before commit.
- `npm run lint`, `npm run typecheck`, `npm run test`, `npm run build`, `npm run qa`, `npm run scan:fake`, and `npm run scan:privacy` pass.
- No push happens without separate operator approval.

## Non-Goals For v0.5

- No real import code.
- No file picker.
- No server-side path resolver.
- No watcher.
- No recursive scan.
- No file copy.
- No OCR.
- No text extraction.
- No summarization.
- No embeddings.
- No vector store.
- No private-vault inspection.
- No upload handling.
- No cloud/API path.
- No GitHub publication of real or derivative content.
