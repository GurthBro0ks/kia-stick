# KIA-Stick-v0.8.6-runtime-ux-reality-audit

Product version: `0.7.0`
Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`
Queue item: `queue-041-v086-runtime-ux-reality-audit`

## Scope

This audit covers repo-owned fake-only runtime surfaces: Chat, Sources, Saved, Upload, Import, Vault, Settings, `/health`, and `/version`.

It does not inspect private folders, name a real document, read a user file, scan a source mount, approve queue-015, approve real-doc work, or add real-document implementation. It does not approve real-doc implementation.

## Evidence

- `components/KiaStickApp.tsx` renders the Chat, Sources, Saved, Upload, Import, Vault, and Settings surfaces from local fake models and browser-local fake metadata.
- `lib/answerGovernor.ts`, `lib/sourceModel.ts`, `lib/savedAnswers.ts`, `lib/importWizardModel.ts`, and `lib/vaultModel.ts` provide deterministic fake data and fake metadata state.
- `app/health/route.ts` reports `fakeOnly: true`, `realDbTouched: false`, product `0.7.0`, and the unchanged prompt version.
- `app/version/page.tsx` reports display, product, channel, build date, git SHA, corpus, index, prompt, and provider.
- `scripts/operator-qa-smoke.mjs` runs static checks and optional loopback-only route checks.

## Findings

1. Chat already blocks save for no-answer responses, but the operator needed clearer visible copy when a no-answer card has context-only fake sources.
2. Saved already stores local fake answer metadata, but the empty state needed to explain no-answer exclusion and version metadata.
3. Sources already groups by hierarchy, but the operator needed denser citable versus context-only labels.
4. Upload already uses fake metadata buttons only, but it needed a scan-friendly cue rail for fake-only QA.
5. Import and Vault already model fake state machines and fake governance, but each needed compact operator QA status cues.
6. Narrow/mobile layout needed stable wrapping for metadata cue rails and bottom nav labels.
7. `/health` and `/version` remain identity surfaces only; no productVersion or promptVersion bump is approved by this phase.

## Highest-value fake-only polish selected

- Improve no-answer and Saved clarity without changing answer generation.
- Add citable/context-only density to Sources.
- Add visible metadata-only QA cues to Upload, Import, and Vault.
- Harden narrow view layout for repeated operator QA.
- Update the smoke helper to list the full fake-only bundle review surface.
