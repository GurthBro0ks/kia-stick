# KIA Stick v0.7.15 Synthetic Packet Report Runner

Phase: `KIA-Stick-v0.7.15-synthetic-packet-report-runner`

Status: synthetic-only report runner for built-in approval-packet fixtures

Product version: `0.7.0`

Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`

## Scope

This phase adds a local report runner for the accepted v0.7.14 synthetic approval-packet validator. It runs built-in synthetic fixtures only and prints a readable GitHub-safe report with `PASS`, `WARN`, and `FAIL` outcomes.

This phase is not a real-doc approval packet. It is not real-doc implementation. It does not approve real-doc work. It does not name, request, read, scan, upload, copy, hash, OCR, index, summarize, redact, inspect, transform, or touch any real document or private source folder.

`queue-015-v07-first-real-doc-gate-request` remains blocked.

## Report Helper Contract

`scripts/synthetic-packet-report.mjs` is exposed as:

```bash
npm run packet:report
```

The helper:

- Uses the existing `lib/syntheticApprovalPacketValidator.ts` validator.
- Runs built-in synthetic fixtures only.
- Prints fake IDs, guard fields, counts, product/prompt versions, queue IDs, and `PASS/WARN/FAIL` labels only.
- Includes `queue015Status: blocked`.
- Includes `realDocumentAccessed: false`.
- Confirms `readsUserFiles: false`, `scansDirectories: false`, and `acceptsPathArguments: false`.
- Rejects unexpected positional arguments.

The helper does not accept arbitrary file path input. It does not read user-provided files. It does not check filesystem existence for packet fields. It does not scan directories.

## Built-In Fixtures

The report includes exactly these fixture classes:

- `fixture-pass-complete`: complete synthetic packet, expected `PASS`.
- `fixture-warn-missing-reviewer-retention`: safe synthetic packet missing reviewer and retention placeholders, expected `WARN`.
- `fixture-fail-broad-private-recursive`: unsafe synthetic packet with broad/private/recursive/source-path wording, expected `FAIL`.

These fixtures use fake labels only. They are not filenames, document titles, hashes, source paths, source directories, real identifiers, snippets, OCR text, uploads, embeddings, indexes, or vector data.

## GitHub-Safe Output Rules

Allowed report output:

- Phase name.
- Product and prompt versions.
- Synthetic fixture IDs.
- Queue ID/status guard values.
- Fake reason codes.
- Counts.
- Booleans.
- `PASS`, `WARN`, and `FAIL` labels.

Forbidden report output:

- Real document names.
- Real snippets.
- Real paths.
- Private source folder names.
- Account/session data.
- Secrets or credentials.
- Hashes from sensitive material.
- OCR output.
- Embeddings or vector data.
- Upload payloads.
- Private notes.

## Rejection Rules

The report runner must fail if a caller provides an unexpected positional argument, including a path-shaped argument. It must not inspect that path, check whether it exists, read it, scan a containing directory, or treat it as a packet source.

The report runner must stay a built-in fixture report. Any future packet-file loader, upload handler, file picker, directory picker, drag/drop import zone, path reader, OCR step, redaction step, text extraction, summarization, embedding, indexing, vector store, APWU/private source access, cloud/API call, service change, or queue-015 transition needs a separate operator-approved phase.
