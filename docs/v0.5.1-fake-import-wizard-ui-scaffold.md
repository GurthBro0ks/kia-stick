- Phase: `KIA-Stick-v0.5.1-fake-import-wizard-ui-scaffold`
- Baseline: accepted v0.5 plan closeout at `bf2248b`
- Scope: fake-only Import Wizard UI scaffold
- Product version impact: none; `productVersion` remains `0.4.0`
- Real document access: none
- Private vault inspected: no
- Push performed: no

## Scope

v0.5.1 implements the accepted v0.5 import wizard plan as a fake metadata UI scaffold only. It adds an `Import` tab with synthetic wizard state, fake audit events, fake proof export, and visible blocked transition reasons.

This phase does not implement real import, file pickers, source path readers, file reads, copying, OCR, text extraction, summarization, uploads, indexing, embeddings, vector stores, or private-vault inspection.

The existing Upload surface remains available as a fake metadata queue, but it now uses explicit fake queue buttons instead of a browser file input.

## Implemented Screens

The Import tab exposes all planned wizard screens:

- Start / Safety
- Source Placeholder
- Scope Confirm
- Copy-to-Quarantine Confirm
- Provenance / Hash Receipt
- Redaction Detection Preview
- Admin Redaction Review
- Metadata Review
- Index Eligibility
- Audit Summary

Each screen uses plain steward-friendly copy and a visible stop sign.

## Fake State Machine

The fake wizard uses synthetic metadata only:

- Fake record ID
- Fake display alias
- Fake source alias
- Fake item count
- Fake hash
- Fake provenance
- Fake redaction categories
- Fake proof ID

Actions move the mock record through the wizard gates. Invalid actions write a visible blocked reason and a fake audit entry. The UI includes blocked test actions for future file picker and skip-to-index attempts.

## Required UI Rules

- Selection is not import.
- Quarantine is not indexable.
- Redaction is not approval.
- Approval is not indexing.

The UI states these rules directly and keeps real-content actions disabled as placeholders.

## Proof Export

The fake proof export includes:

- Build identity
- Current fake wizard step
- Fake metadata record
- Fake audit events
- Guard flags proving fake-only scope

The proof export excludes:

- Raw documents
- Real filenames or private paths
- File bytes
- OCR text
- Snippets
- Uploads
- Vector-store data
- Secrets

## Tests

Vitest coverage checks:

- all planned wizard screens are present
- the UI renders stop-sign labels
- no file input exists in the Import Wizard panel
- fake records move through the full wizard path
- skipped gates are blocked with visible reasons
- real path/content payloads are rejected and sanitized from audit notes
- fake proof exports include build identity and no private paths

## Boundary

Future phases remain fake-only unless separately authorized. This scaffold does not approve real import implementation.
