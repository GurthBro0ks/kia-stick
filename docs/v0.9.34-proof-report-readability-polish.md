# KIA-Stick-v0.9.34-proof-report-readability-polish

Target machine: `USER_LAPTOP_ONLY`

## Scope

This phase improves fake-only proof/report readability so accepted WARN is easy to distinguish from PASS and FAIL. It does not change dependency state, runtime security state, or real-doc capability.

## Readability Rules

- `RESULT=PASS` remains the only clean proof result.
- `RESULT=WARN` with explicit `ACCEPTED_WARN` or `OPERATOR_QA_ACCEPTED_WARN` is shown as `ACCEPTED_WARN_PARKED`.
- Accepted WARN is not treated as PASS.
- Unaccepted WARN remains `WARN_REVIEW_REQUIRED`.
- FAIL remains a stop condition.
- Missing `RESULT.md` remains `WARN_MISSING_RESULT`.

## Proof Fields That Must Stay Visible

- `RESULT`
- `WARN_REASON` when present
- manual QA status
- pushed state
- dirty state
- package-lock unchanged
- `queue-015-v07-first-real-doc-gate-request` blocked
- `KIA-Stick-v0.9.12C-next-runtime-framework-security-implementation` blocked
- Next/PostCSS parked
- real-doc capability blocked

## Helper Changes

- `scripts/local-proof-index.mjs` adds a `review_state` label to CLI and Markdown output.
- `scripts/closeout-helper.mjs` reports `proof_accepted_warn=yes` and `ACCEPTED_WARN_STATUS=accepted_warn_parked` for explicit accepted-WARN proof.
- Closeout review still returns WARN for accepted-WARN proof, because the unresolved Next target is parked rather than fixed.

## Safety

- Helpers read only the already allowed proof metadata surfaces.
- No helper reads private paths, `.env` files, screenshots, real docs, arbitrary user files, or source document content.
- No package mutation, Next target selection, PostCSS override, or runtime implementation is approved.
