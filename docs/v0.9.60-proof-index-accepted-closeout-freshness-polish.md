## Proof Index Polish

`scripts/local-proof-index.mjs` now separates proof freshness labels instead of relying on the old ambiguous `Latest review-ready proof` line.

Expected `latest` output includes:

- `Latest proof:`
- `Latest accepted pushed closeout proof:`
- `Latest operator QA PASS proof:`
- `Latest accepted-WARN proof:`
- `Latest screenshot review-ready candidate:`
- `Latest review-ready proof:` as a legacy-compatible screenshot-gated label
- `Review-ready candidate criteria:`

The accepted pushed closeout selector recognizes proof-safe nested closeout directories such as `closeout_push_20260630T204915Z` under the supplied local proof root and reads only `RESULT.md` metadata. It does not require screenshots, does not mutate proof directories, and does not inspect arbitrary folders outside the supplied root.
