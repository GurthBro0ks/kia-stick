# KIA-Stick-v0.9.39-closeout-summary-proof-dir-usability

Target machine: `USER_LAPTOP_ONLY`

## Scope

This phase improves `scripts/closeout-helper.mjs` so closeout review and summary output can be tied to a supplied proof directory instead of always using latest-proof discovery.

## Explicit Proof Directory Behavior

Supported operator forms:

- `npm run closeout:summary -- --proof-dir <path>`
- `npm run closeout:review -- --proof-dir <path>`
- `npm run closeout:summary -- --proof-root /home/mint/kia-stick-local-proofs`

When `--proof-dir` is supplied, output includes `proof_discovery_mode=explicit_proof_dir`, the supplied proof path, proof result, manual QA status, and pushed state parsed from that proof's `RESULT.md`.

When no proof dir is supplied, output includes `PROOF_DISCOVERY_MODE=default_latest_from_proof_root` and a note that default discovery uses the latest proof under the proof root and may be generic.

## Gate Behavior

- WARN and FAIL proof rules remain active.
- Accepted-WARN remains parked WARN, not PASS.
- Real proof problems are still reported.
- The helper does not execute `git push`.
- The helper reads only `RESULT.md` from a proof directory under `/tmp` or `/home/mint/kia-stick-local-proofs`.

## Boundary

This is proof-helper usability work only. It adds no dependency change, runtime Next/PostCSS implementation, file picker, FileReader, upload, OCR, indexing, vector store, or real-doc access.
