## Ranked Issues And Opportunities

1. **Chat/Saved handoff clarity**
   - Confidence: `0.90`
   - Why it matters: Chat and Saved form the most-used fake-only workflow. The user needs clear proof that a fake answer was saved, that version metadata traveled with it, and that no-answer states do not create misleading saved records.
   - Recommended action: Add copy/state hardening and tests for answer save feedback, saved empty/detail states, duplicate save handling, and saved metadata display.

2. **Upload/Import fake-only guard consistency**
   - Confidence: `0.86`
   - Why it matters: Upload and Import are the highest-risk UX labels because they resemble real file flows. They need to stay button-only, fake-metadata-only, and visually consistent about blocked real actions.
   - Recommended action: Align fake confirmation text, blocked-action reasons, and proof-export guard language. Add deterministic tests that the fake-only controls still have no file input path.

3. **Vault workflow scan density**
   - Confidence: `0.80`
   - Why it matters: Vault rows carry many fake governance fields. The UI should make workflow state, redaction status, index eligibility, and audit safety quick to scan.
   - Recommended action: Tighten row labels and summary counts after the Chat/Saved/Upload pass, not in the first implementation chunk.

4. **Sources/version confidence**
   - Confidence: `0.78`
   - Why it matters: Sources and version surfaces are the proof spine. Manual QA should be able to confirm that fake citations, source hierarchy, `/health`, and `/version` agree.
   - Recommended action: Keep as checklist coverage in v0.7.4; defer UI changes unless QA finds mismatch.

5. **Settings copy drift**
   - Confidence: `0.74`
   - Why it matters: Settings repeats fake-only and version claims. It can drift after runtime identity changes.
   - Recommended action: Include Settings in manual QA and only patch copy when mismatch is found.

## Explicit Non-Approval

This plan does **not** approve real-doc work.

Do not add:

- File pickers or directory pickers.
- Path readers or filesystem reads for documents.
- Upload handlers for real documents.
- OCR, text extraction, summarization, transforms, embeddings, indexing, or vector stores.
- Real redaction, real metadata review, or real quarantine copy.
- Private-vault inspection.
- Services, cron, timers, Caddy, DNS, or secret handling.
- Any proof artifact containing raw document text, account/session data, identifiers, private paths, snippets, hashes from sensitive material, captures, or proprietary originals.

- Rerun release, QA, fake scan, privacy scan, closeout review, and diff checks before any replacement push.
