# KIA-Stick-v0.8.7-chat-saved-no-answer-polish

Product version: `0.7.0`
Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`
Queue item: `queue-042-v087-chat-saved-no-answer-polish`

## Scope

This subphase improves fake-only Chat and Saved ergonomics around no-answer states, save status, citations, version identity, and operator clarity.

Allowed runtime file touched: `components/KiaStickApp.tsx`.

## Runtime polish

- No-answer assistant cards now show `Unsaved no-answer` instead of implying a normal save path.
- A no-answer save attempt records a visible warning notice: no-answer responses stay out of Saved and the operator should review the context-only fake trail.
- Chat composer status now says `No-answer unsaved` after the latest answer has no controlling fake source.
- Saved empty state now explains that only cited fake Chat answers become Saved records and no-answer cards are intentionally excluded.
- The no-answer empty-state copy points the operator to the full packet for context-only fake source review.

## Preserved behavior

- Deterministic fake answer behavior is unchanged.
- Saved records still use `createSavedAnswerRecord` and `upsertSavedAnswer`.
- Saved records still include product version, prompt version, build label, provider, mode, scope, detail, timestamp, and citation count.
- No-answer records are not inserted into Saved.

## Safety boundary

This subphase uses fake Chat/Saved state only. It does not add real documents, file choosers, path readers, uploads, OCR, indexing, vector stores, real redaction, private folder inspection, or queue-015 unblocking.
