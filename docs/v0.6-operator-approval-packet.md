# KIA Stick v0.6.4 Operator Approval Packet

Phase: `KIA-Stick-v0.6.4-operator-approval-packet`

Status: `PLAN ONLY`

Baseline: `origin/main` verified at `bc8c9df`, the pushed v0.6.3 fake-only pilot simulator.

Product version impact: none. `productVersion` stays `0.4.0`; any future product milestone bump must be approved in a separate implementation phase. This packet only documents that a future bump would be future-prompt-only.

Model recommendation: Claude for the approval and safety wording. GPT/Codex is acceptable for docs and tests. GLM-5.2 is optional for a second-opinion review only.

## Purpose

This document is a copy-ready operator approval packet **template** for any future one-document real-doc pilot. An operator fills in the fields below, classifies every gate, signs, and dates it. A completed packet is a precondition for review only.

A completed and signed packet, by itself, **does not authorize implementation**. Any later implementation prompt must separately name exactly one gate and exactly one document, and must still pass `docs/v0.6-real-doc-safety-checklist.md`.

## Plan-Only Boundary

This packet is a planning and approval artifact only. It does not authorize or implement real-document reads, directory listing, file selection, file pickers, path reading, copying, OCR, text extraction, summarization, transformation, embeddings, indexing, vector storage, upload handling, private-vault inspection, exports, service changes, pushes, or private-data commits.

No real document path, filename, text, snippet, image, OCR output, identifier, account data, session data, hash, export, vector data, or private note is needed to complete this phase, and none may be pasted into a completed packet.

The only approved output of this phase is GitHub-safe docs, tests, queue state, and closeout state.

## How To Use This Packet

1. Copy the template below into a private working note. Do not commit a filled-in packet that contains private details to Git.
2. Fill every required field. A blank required field is an automatic `do not proceed`.
3. Classify every gate as `PASS`, `WARN`, or `FAIL`.
4. Sign and date the operator signature block.
5. Re-run `docs/v0.6-real-doc-safety-checklist.md` against the same scope.
6. Only after all gates are `PASS`, hold the packet for a later implementation prompt that names one gate and one document.

## Required Packet Fields

A completed packet must specify all of the following. Any missing field is a `do not proceed` blocker.

### 1. Exact Phase

- Phase name (must be the exact future phase string, not inherited from a prior run).
- Target repo.
- Target machine.
- Baseline `origin/main` short SHA the work starts from.
- Date the approval was written (approval must be fresh and tied to this exact phase).

### 2. One-Document Scope

- A single operator-selected local source item, referenced only by an opaque pilot ID in any GitHub-safe output.
- Explicit statement that exactly one document is in scope.
- Explicit block on batch expansion, companion files, attachments, archives, and derived exports.
- Explicit non-recursive rule: no directory listing, recursive traversal, tree scan, folder picker, globbing, archive expansion, or auto-discovery.

### 3. Allowed Actions

List only the narrow, reversible actions the operator approves for this one gate, for example:

- The single gate to be exercised (named explicitly).
- The single document referenced by opaque pilot ID.
- GitHub-safe proof generation using labels, counts, booleans, and command names only.

### 4. Blocked Actions

Explicitly block every action not named in Allowed Actions, including at minimum:

- File picker, path reader, file open, file read.
- Copy, OCR, text extraction, summarization, transformation.
- Embeddings, indexing, vector store, upload handling.
- Export, backup, private database write.
- Service change, push, private-data commit.
- Any action beyond the one named gate.

### 5. Rollback

- A written rollback for the named gate that can delete any private working artifact before any retry.
- Confirmation that the real source is never mutated.
- Confirmation that a retry never reuses unsafe artifacts.

### 6. Deletion And Retention

- Retention defaults to deletion of any private working artifact.
- If retention is approved, the packet names a private retention period, an owner, and a review date.
- Private artifacts are never committed, exported, uploaded, or indexed by default.

### 7. GitHub-Safe Proof

- Proof uses only allowed fields (see GitHub-Safe Proof Rules below).
- Proof never contains raw text, snippets, private paths, identifying filenames, OCR text, sensitive hashes, exports, vector data, or private notes.
- Proof includes `real_document_access=false` for any planning phase.

### 8. Operator Signature And Date

- Operator name or initials.
- Signature line.
- Date.
- Explicit statement: "I approve the named gate for the named single document only, and I understand this packet does not authorize implementation by itself."

## Approval Gate Checklist

Before any future real content is touched, the operator must classify every gate below as `PASS`, `WARN`, or `FAIL`.

- `PASS`: the field/gate is explicit, narrow, reversible, and GitHub-safe proof can be produced without exposing private details.
- `WARN`: the field/gate has an ambiguity that must be resolved in writing before the next content-touching action.
- `FAIL`: the field/gate is missing, too broad, not reversible, or would require private content in proof.

Any `FAIL` is a **do not proceed** blocker. Any unresolved `WARN` is also a **do not proceed** blocker for content-touching work.

| Gate | PASS | WARN | FAIL |
| --- | --- | --- | --- |
| Exact phase | Phase, repo, machine, baseline SHA, and approval date are present and tied to this exact run. | One identifier is present but ambiguous or undated. | Phase, repo, machine, or baseline is missing, generic, or inherited from a prior run. |
| One-document scope | Exactly one operator-selected source is in scope, referenced by opaque pilot ID, with batch and recursion blocked. | One document is intended but companion files, attachments, or derived exports are ambiguous. | Any folder, batch, archive, recursive scan, auto-discovery, mount scan, cloud source, or upload source is requested. |
| Allowed actions | Only narrow, reversible, named actions for one gate are listed. | Allowed actions are listed but one action lacks a reversibility statement. | Allowed actions are broad, open-ended, or exceed the one named gate. |
| Blocked actions | Every non-approved action family is explicitly blocked. | Blocked list exists but omits one action family. | Blocked list is missing, or an unsafe action is allowed by implication. |
| Rollback | Rollback is written for the named gate and can delete private working artifacts before retry. | Rollback exists but one artifact class lacks an operator decision. | No rollback exists, the source could be mutated, or retry would reuse unsafe artifacts. |
| Deletion and retention | Retention defaults to deletion unless an approved private retention period, owner, and review date are named. | Retention label exists but owner or review date is missing. | Private artifacts are retained by default, committed, exported, uploaded, or left without deletion instructions. |
| GitHub-safe proof | Proof uses only safe labels, counts, booleans, and command names, with `real_document_access=false` for planning. | Proof fields are mostly safe but one field is unclassified. | Proof would require raw text, snippets, private paths, identifiers, OCR text, sensitive hashes, exports, or vector data. |
| Operator signature and date | Signature, name/initials, date, and the no-implementation acknowledgement are present. | Signature exists but date or acknowledgement wording is missing. | Signature, date, or acknowledgement is missing, stale, or copied from a prior phase. |

## Do-Not-Proceed Blockers

Stop before any real content is touched if any item is true:

- Any required packet field above is blank, generic, inherited, or undated.
- Operator approval is stale, implied, pasted from an older run, or not tied to the exact phase.
- The source scope includes a folder, batch, archive, export set, recursive rule, auto-discovery, synced location, cloud source, or private repository.
- The requested action includes file picker, path reader, file open, copy, OCR, extraction, summarization, transformation, embeddings, indexing, upload handling, vector store, export, service change, push, or private-data commit before the one named gate is approved.
- Rollback, deletion, or retention is unwritten.
- The repo has unrelated dirty changes, untracked private artifacts, or tracked private storage paths.
- Proof would need raw text, snippets, private paths, identifiers, OCR text, sensitive hashes, exports, vector data, or private notes to justify the result.
- The signature block is unsigned or undated.

## GitHub-Safe Proof Rules

Allowed proof fields:

- Phase name.
- Commit SHA.
- Command names.
- PASS/WARN/FAIL states.
- Gate names and outcome labels.
- Counts by safe category.
- Opaque pilot ID.
- Booleans such as `real_document_access=false` for planning phases.

Forbidden proof fields:

- Raw document text.
- Snippets.
- Private paths.
- Filenames when identifying.
- Member, employee, witness, officer, case, account, session, device, contact, or credential identifiers.
- OCR text.
- Hash values when sensitive.
- Exports.
- Backups.
- Private databases.
- Vector data.
- Embeddings.
- Private notes.
- Screenshots of real content.
- Any field that lets a reviewer reconstruct or locate the private source.
