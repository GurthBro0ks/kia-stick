# KIA Stick v0.7.14 Synthetic Approval-Packet Validator

Phase: `KIA-Stick-v0.7.14-synthetic-approval-packet-validator`

Status: synthetic-only approval-packet validator for future one-document / one-gate review packets

Product version: `0.7.0`

Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`

## Scope

This phase adds deterministic docs/tests/tooling for fake packet objects only. It validates whether a future review packet shape is complete enough to return `PASS`, incomplete but safe enough to return `WARN`, or unsafe enough to return `FAIL`.

This is not an approval packet. It is **not** a real-doc approval packet, and it does not approve real-doc work. It does not name, read, scan, upload, copy, hash, OCR, index, summarize, redact, inspect, transform, or touch any real document or private source folder.

`queue-015-v07-first-real-doc-gate-request` remains blocked.

## Validator Contract

`lib/syntheticApprovalPacketValidator.ts` validates in-memory synthetic packet objects only.

- No file path arguments.
- No filesystem existence checks.
- No directory scans.
- No user-supplied file reads.
- No path, source folder, private source, recursive, wildcard, or broad source input is accepted.
- No real document title, file name, source mount, source path, account/session value, token, hash, OCR text, snippet, embedding, vector data, or upload payload is accepted.

The validator returns:

- `PASS`: the packet is complete, synthetic-only, and contains no unsafe scope wording.
- `WARN`: the packet is safe but incomplete, such as a missing reviewer placeholder or deletion/retention placeholder.
- `FAIL`: the packet is unsafe, path-shaped, broad, private, recursive, source-path-shaped, real-doc-capability-shaped, has the wrong number of placeholders, or implies `queue-015` is unblocked.

## Required Synthetic Fields

A complete synthetic packet must include:

- One fake packet ID.
- Exactly one future document placeholder.
- Exactly one future gate placeholder.
- One narrow synthetic allowed action.
- Blocked actions.
- Rollback plan.
- Deletion/retention plan.
- Reviewer placeholder.
- Proof-safe output agreement.
- Stop-on-WARN/FAIL agreement.
- `queue015Status=blocked`.
- `queue015BlockedConfirmation=true`.
- `realDocumentAccessed=false`.

The completeness check requires exactly one future document placeholder and exactly one future gate placeholder.

Exactly one future document placeholder means one synthetic label such as `SYNTH-ONE-DOC-ALPHA`. It is not a filename, path, real title, hash, or document identifier.

Exactly one future gate placeholder means one synthetic label such as `SYNTH-GATE-METADATA-ONLY`. It is not implementation approval.

## Rejection Rules

These values return `FAIL`:

- Absolute paths, home paths, mounted-source paths, relative paths, or file-extension-shaped values.
- APWU or USPS source markers.
- `private`, `real-documents`, source-path, source folder, directory, recursive, batch, wildcard, or broad "all/every/entire" wording.
- File picker, directory picker, drag/drop import, path reader, file read, upload, OCR, indexing, embedding, summarization, vector-store, or real-redaction requests outside the blocked-action list.
- More than one document placeholder.
- More than one gate placeholder.
- Missing blocked actions, rollback plan, proof-safe agreement, stop-on-WARN/FAIL agreement, or queue-015 blocked confirmation.
- Any text implying `queue-015` is ready, accepted, open, unblocked, or clear to proceed.

Missing reviewer or deletion/retention placeholders return `WARN` if every unsafe-field check still passes.

## Synthetic Examples

`PASS` example:

- Packet ID: `FAKE-APPROVAL-PACKET-001`
- Document placeholder: `SYNTH-ONE-DOC-ALPHA`
- Gate placeholder: `SYNTH-GATE-METADATA-ONLY`
- Allowed action: `SYNTH-METADATA-LABEL-CHECK-ONLY`
- Blocked actions: synthetic block IDs only
- `queue015Status=blocked`
- Proof-safe and stop-on-WARN/FAIL agreements are true

`WARN` example:

- Same synthetic packet as above, but reviewer and deletion/retention placeholders are not filled in yet.
- No path, real-doc, private, recursive, source-path, upload, OCR, indexing, vector, or queue-015 unblock language appears.

`FAIL` example:

- Any packet with a path-shaped value, broad source request, recursive request, private/source marker, more than one document, more than one gate, or any queue-015 unblock implication.
