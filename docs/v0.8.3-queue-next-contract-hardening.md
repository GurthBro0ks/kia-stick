# KIA Stick v0.8.3 Queue Next Contract Hardening

Phase: `KIA-Stick-v0.8.3-queue-next-contract-hardening`

Status: queue tooling contract hardening

Product version: `0.7.0`

Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`

## Scope

This phase hardens `npm run queue:next` so stale historical options, blocked gates, or parked work do not hide the true next actionable task.

The implementation is limited to queue docs/tests/tooling and queue state. It does not change runtime UI, runtime capability, product version, prompt version, privacy boundaries, or git push behavior.

## Contract

`queue:list` remains a complete audit listing. It still prints accepted, blocked, parked, and actionable queue items.

`queue:set` remains the explicit mutation command and keeps the existing private-path and secrets-looking text guards.

`queue:next` is now an actionable selector:

- It skips `accepted` items because they are closed.
- It skips `blocked` items because they require an external approval or safety condition before work can start.
- It skips `parked` items because they are intentionally deferred or superseded.
- It selects the first remaining item in queue order.

Actionable statuses are therefore `planned`, `running`, `needs_review`, and `ready_to_push`. Queue order still matters.

## Expected Result

After the v0.8.1 and v0.8.2 reconciliation, `queue-015-v07-first-real-doc-gate-request` remains blocked and is skipped by `queue:next`. The next selected item is the current review-pending bundle work, starting with `queue-036-v081-queue-reality-audit`.

## Safety Boundary

This phase does not add file pickers, directory pickers, drag/drop import zones, path readers, uploads, OCR, real redaction, text extraction, summarization, embeddings, indexing, vector stores, upload handlers, private-source inspection, services, secrets, skills, global config changes, or push behavior.

## Clear Answer

`queue:next` now reports the next actionable queue item instead of the first merely non-accepted item.
