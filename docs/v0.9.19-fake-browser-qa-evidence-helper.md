# KIA-Stick-v0.9.19-fake-browser-qa-evidence-helper

Target machine: `USER_LAPTOP_ONLY`

## Scope

This phase adds a no-new-dependency local helper for repeatable fake-only browser QA evidence planning and export. It is deterministic checklist/report tooling, not browser automation.

## Helper

`scripts/fake-browser-qa-evidence.mjs` produces proof-safe Markdown and JSON for these surfaces:

- Chat.
- Sources.
- Saved.
- Upload.
- Vault.
- Import.
- Settings.
- `/version`.
- `/health`.

The helper reads repo-owned metadata only: `package.json`, `feature_list.json`, and `docs/phase-backlog.json`. It can write output only under `/home/mint/kia-stick-local-proofs` or `/tmp`.

## Browser Automation Status

Browser automation status: `manual_checklist_export`. Automated browser capture is not added in this bundle because repeatable proof output is safer as a deterministic local checklist/export helper.

Manual operator evidence can still be attached later by opening the generated checklist and recording screenshots in a separately reviewed proof directory.

## Fake-Only Boundaries

- No real documents are read, listed, copied, summarized, uploaded, indexed, OCR processed, or embedded.
- No file picker, FileReader, upload handler, path reader, directory picker, drag/drop import zone, vector store, browser File object, or private-source access is added.
- Output is proof-safe state metadata and surface checklist text only.
- Product/package version remains `0.7.0`.
- Prompt version remains `prompt.fake-docs.v0.5-import-wizard-hardening`.
- `queue-015-v07-first-real-doc-gate-request` remains blocked.
