## Scope

v0.3 implements private-vault governance screens using synthetic metadata fixtures in `lib/vaultModel.ts`.

It does not read, copy, scan, OCR, index, summarize, transform, upload, or commit real APWU, USPS, member, local, case, steward, or private-vault documents.

- `/media/mint/SHARED/APWU`

## UI Surfaces

The new Vault tab contains these governance surfaces:

- Vault: lane counts and lifecycle state machine.
- Quarantine: fake intake placeholders that are never indexable.
- Redaction Review: fake detection flags and review decisions.
- Metadata Review: authority, source status, sensitivity, redaction status, metadata status, reviewer, and hash fields.
- Index Eligibility: explicit eligibility states and reason codes.
- Audit Log: local mock transition log.

Existing Chat, Sources, Saved, Upload, Settings, `/health`, and `/version` flows remain present.

## Lifecycle

Each fake metadata record can move through this local mock state machine:

```text
selected
-> quarantine
-> hash/provenance
-> redaction detection
-> admin review
-> approved redacted copy
-> metadata
-> index decision
-> audit
```

The fake action button advances local mock state only. It does not open, read, copy, hash, or inspect any real file.

## Index Rules

The UI repeats the core warning:

```text
Quarantine, redaction, and admin review are governance gates. They do not make an item indexable.
```

Eligibility remains explicit:

- `not_eligible`
- `eligible_private`
- `eligible_redacted`
- `eligible_public`

Quarantine and restricted-sensitive lanes are not indexable by default.

## Guard and Tests

`lib/vaultModel.ts` includes `assertFakeMetadataOnly` and `applyVaultAction`.

The guard blocks action payloads with:

- real/private boundary references
- path-like fields
- file/content/raw-text fields
- browser `File` objects

Tests cover:

- all lanes have fake metadata fixtures
- lifecycle advancement creates audit entries
- quarantine remains not indexable
- private paths and raw content fields are blocked

## GitHub Safety
