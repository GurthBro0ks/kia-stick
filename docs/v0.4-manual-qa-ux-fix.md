# KIA Stick v0.4 Manual QA UX Fix

- Phase: `KIA-Stick-v0.4-manual-qa-ux-fix`
- Target: `USER_LAPTOP_ONLY`
- Status: implemented using fake metadata only
- Real document access: none
- Private vault inspected: no
- Push performed: no

## Fixes

Manual QA found that v0.4 was functionally hardened but rough to operate. This pass fixes those blockers without adding real-document handling.

## Saved Answers

Saved answers now use a stable same-chat key built from question, mode, scope, answer identity, and citations.

- Saving a new answer creates one saved card.
- Saving the same unchanged answer shows `Already saved. No new data.`
- Saving the same answer after metadata or detail changes replaces the existing card.
- A genuinely different question or answer context creates a separate saved card.

## Citations

Chat citations are collapsed by default behind `Show citations (n)`. Expanding the button shows the citation cards. This keeps the answer readable on mobile while preserving citation-first review.

## Settings

The Settings tab uses extra bottom spacing so the fixed bottom toolbar remains visible and usable on mobile.

## Vault Guide Mode

The Vault tab now starts with a plain-English guide:

- What this screen means
- What is safe
- What is blocked
- What happens next

Technical lifecycle rails, lane counts, workflow counts, detailed field grids, fake refs, provenance, and redaction flags remain available behind `Show technical details`.

## Tests

Vitest coverage now checks:

- saved-answer creation
- duplicate-save warning support
- changed same-chat replacement
- separate cards for changed answer context
- citation collapse default state
- bottom navigation markup including Settings
- Vault guide copy
- technical details hidden by default

## Boundary

This phase stays fake-doc/fake-metadata-only. It does not read, list, copy, scan, OCR, ingest, summarize, transform, upload, index, or commit real APWU, USPS, member, local, case, steward, or private-vault documents.
