# KIA Stick v0.7.11 Persistent Proof Index and Review Guide

Phase: `KIA-Stick-v0.7.11-persistent-proof-index-and-review-guide`

This phase adds local-only proof discoverability for persistent proof bundles. It is fake-proof metadata tooling only.

## Persistent Proof Locations

- Accepted persistent proof root: `/home/mint/kia-stick-local-proofs`
- Desktop pointer file: `/home/mint/Desktop/kia-stick-proofs/LATEST_KIA_PROOF.txt`
- Latest accepted v0.7.10b proof: `/home/mint/kia-stick-local-proofs/proof_kia_stick_v0_7_10b_operator_smoke_evidence_20260626T090618Z`

`/tmp` proof folders are useful for validation runs, but `/tmp` alone is not enough for final operator acceptance. Final acceptance should point to a persistent proof folder under `/home/mint/kia-stick-local-proofs` and should preserve a desktop pointer that an operator can inspect later.

## Local Helper

Run:

```bash
npm run proof:index
npm run proof:index -- latest
npm run proof:index -- write
```

The helper defaults to `/home/mint/kia-stick-local-proofs`. It lists KIA proof directories newest-first, reports the newest proof, reports the newest review-ready proof, shows whether `RESULT.md` and `OPEN_THIS_FOLDER.txt` exist, and counts image files under a proof's `screenshots/` folder.

A review-ready proof is a proof with `RESULT.md`, `RESULT=PASS`, `OPEN_THIS_FOLDER.txt`, and at least one screenshot. This separates a newer validation-only proof from the latest operator-review proof when both live under the persistent proof root.

The `write` command generates:

- `/home/mint/kia-stick-local-proofs/LOCAL_PROOF_INDEX.md`
- `/home/mint/kia-stick-local-proofs/LOCAL_PROOF_INDEX.json`

Missing `RESULT.md` is reported as `WARN_MISSING_RESULT`. A missing or incomplete proof directory must not be silently accepted.

## Operator Review Steps

1. Open `/home/mint/kia-stick-local-proofs`.
2. Confirm the latest accepted v0.7.10b proof appears in the helper output.
3. Confirm `/home/mint/Desktop/kia-stick-proofs/LATEST_KIA_PROOF.txt` points to the accepted proof folder.
4. Open the proof folder's `RESULT.md`.
5. Open the proof folder's `OPEN_THIS_FOLDER.txt` if present.
6. Visually review the screenshots under `screenshots/`.
7. Treat missing `RESULT.md`, missing screenshots, or unreadable proof folders as review blockers.

## Safety Boundary

The helper does not read arbitrary home folders. Custom roots are restricted to the accepted local proof root or temporary fixture/proof directories.

The helper reads only:

- proof directory names
- `RESULT.md`
- `OPEN_THIS_FOLDER.txt`
- filenames under `screenshots/`

It does not read real documents, private vault contents, APWU/private source paths, app upload payloads, OCR output, embeddings, vector stores, or indexed source data. It does not add file pickers, directory pickers, drag/drop zones, path readers, runtime uploads, services, global skills, secrets, cron, timers, tmux, Caddy, DNS, or Discord notification behavior.
