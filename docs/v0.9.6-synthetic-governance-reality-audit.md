# KIA-Stick-v0.9.6-synthetic-governance-reality-audit

Product version: `0.7.0`
Prompt version: `prompt.fake-docs.v0.5-import-wizard-hardening`
Queue item: `queue-051-v096-synthetic-governance-reality-audit`

## Purpose

This audit records the current synthetic governance surface after the accepted v0.9.1 through v0.9.5 proof durability closeout.

This is synthetic-only docs/tests/tooling/state work. It does not change runtime UI, runtime capability, productVersion, promptVersion, or real-doc approval.

## Inventory

- `lib/syntheticApprovalPacketValidator.ts` validates in-memory synthetic packet objects and returns `PASS`, `WARN`, or `FAIL`.
- `lib/syntheticPacketFixtures.ts` owns the built-in fixture matrix; it does not accept path arguments or load packet files.
- `scripts/synthetic-packet-report.mjs` prints fixed synthetic packet report output from built-in fixtures only.
- `scripts/synthetic-governance-report.mjs` summarizes the fixture matrix, packet report, and packet guard for operator review.
- `scripts/synthetic-packet-safety-guard.mjs` scans fixed allowlisted repo files for synthetic-only drift.
- `scripts/proof-index.mjs` and `scripts/local-proof-index.mjs` provide proof discovery and proof durability metadata.
- `scripts/closeout-helper.mjs` reviews proof, git, and queue state without mutating queue status or pushing.
- `scripts/task-queue.mjs` keeps accepted, blocked, and parked items out of `queue:next`.

## Coverage Gaps Found

- Negative fixture coverage needed clearer cases for missing operator signature, too-broad scope, recursive wording, private/path-shaped wording, multiple documents, multiple gates, missing rollback, missing deletion/retention, missing redaction-policy result, and unsafe proof output.
- Report output needed a visible stop-on-WARN/FAIL cue for operator closeout.
- Closeout helper output needed explicit `stop_on_warn_fail` and `queue_acceptance_allowed` fields.
