# v0.4 True Threaded Chat

Phase: `KIA-Stick-v0.4-true-threaded-chat`
Target: `USER_LAPTOP_ONLY`
Provider: `local-fake-deterministic`

## Scope

This phase replaces the remaining single-question answer workflow with a real fake-only threaded chat.

## Implemented

- Added explicit `ConversationThread`, `ChatMessage`, `UserMessage`, and `AssistantMessage` types.
- Added stable `thread-*`, `turn-*`, and `msg-*` identifiers that are not timestamps alone.
- Stored an ordered user/assistant message list instead of one current question and one current answer.
- Added browser persistence for the current fake thread under a separate key from Saved Answers.
- Added `New chat` with confirmation when the active thread has messages.
- Changed composer placeholder to `Message KIA Stick...`.
- Changed the primary composer action to `Send`.
- Disabled blank sends and prevented double-send while a turn is processing.
- Cleared the composer immediately after send.
- Added Enter-to-send and Shift+Enter newline behavior.
- Added loading and retryable failure assistant rows.
- Moved Save onto each assistant message so saving targets a specific answer.
- Preserved saved-answer dedupe and duplicate warning behavior.
- Kept full packet and citations collapsed per assistant message.
- Added recent-thread context to the deterministic fake answer governor.
- Added supported fake follow-up handling for evidence, verbal denial, supervisor wording, and next steps.
- Added clarifying response behavior when a follow-up has no resolvable prior topic.

## Safety

- Fake documents and fake metadata only.
- No real APWU/USPS/member/local/case/steward documents were read, copied, indexed, summarized, OCRed, uploaded, or committed.
- `/media/mint/SHARED/APWU` was not accessed.
- `~/kia-stick-private-vault` was not inspected.
- Saved Answers remain separate from current-thread persistence.
